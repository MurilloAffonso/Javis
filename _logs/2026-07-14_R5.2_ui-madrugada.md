# R5.2 — Interface: painel da Madrugada no Command Center

**Data:** 2026-07-14
**Fase:** R5.2 (primeira UI de uma capacidade do executor)

## O gap

O executor supervisionado tinha view (`exec.js`), mas a **Madrugada só existia no
CLI**. Quem não abre terminal não tinha como ver se ela estava armada, dentro da
janela, ou com task esperando.

## Decisão de design: read-only + kill switch, sem botão de "executar"

O painel **não tem botão de rodar a Madrugada**. Isso é deliberado e coerente com
o que foi decido em R4.5: um processo que roda código sozinho merece ser
explícito, não um clique de web. Então a UI faz só duas coisas:

1. **Mostra o preflight** (read-only): habilitada?, dentro da janela?, kill
   switch ativo?, quantas tasks aprovadas, qual a task alvo.
2. **Oferece o kill switch.** Desarmar é a direção *segura* (impede execução);
   está na UI. Rodar continua só no CLI.

## Implementação

**Backend** (`server.py`):
- `GET /madrugada/status` — token local + `NightMode().preflight()`, nunca executa
- `POST /madrugada/kill-switch` — token local, cria/remove o arquivo-sentinela

**Frontend** (`js/views/madrugada.js`):
- Nova view no Command Center, item "Madrugada" no NAV, poll a cada 5s
- Botão desarmar/rearmar; sem botão de execução

**Fix de fundo** (`night_mode.py`):
- `NightMode.__init__` resolvia `kill_switch`/`log_dir` no *default da assinatura*
  (vinculado na definição da classe). Trocado para resolução em **tempo de
  execução**, para que um monkeypatch do módulo alcance instâncias criadas sem
  argumento — que é o caso da rota. Sem essa mudança a rota e os testes leriam
  paths divergentes.
